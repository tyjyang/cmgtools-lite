# configuration file for systematics and groups to define in the datacard
# the actual fields might be simplified, this was taken from a previous code
#
# this is digested by cardMaker.py
# first field is a dummy name for user, second one is a regular expression to match the histograms for those systematics (they must exist for shape type, but not for lnN, see later), third field is a regular expression for the processes to assign the syst to (you may also have all histograms but decide to use some systs only for some processes), fourth field is the name of the group appearing in the datacard (the mass one must be declared as SKIPGROUP), fifth field is a keyword for the type of systematics (usually shape or lnN, but shapeNoConstraint for the mass shift), the 6th field is optional, it is used to rescale some systematics in the datacard (when type is shape) or to assign a value (for lnN type, which require no histogram to exist) 
#
# second field is actually not used for lnN nuisances (might be used to trigger some behaviour, e.g. decorrelating by charge if needed)
